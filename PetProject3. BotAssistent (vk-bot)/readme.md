ПРОЕКТ "БОТ-ПОМОЩНИК ПРЕПОДАВАТЕЛЯ"


...:::  О ПРОЕКТЕ  :::...

Цель создания чат-бота:
	Поддержка пользователей – студентов первого курса, обучающихся по дисциплине «Информатика» (информирование, ответы на часто задаваемые вопросы, и т.п.).	

Задачи чат-бота:
Бот берет на себя большую часть переписки со студентами, позволяя преподавателю заняться более важными вещами. 
Бот может в переписке со студентом:
	1) Давать случайную задачу для тренировки
	2) Сообщить дату экзамена 
	3) Сообщить студенту его текущую успеваемость (список зачтенных лаб.работ с разбивкой по темам)
	4) Предоставить связь с преподавателем



...:::  НАЧАЛО РАБОТЫ  :::...

Для работы с ботом необходимо предварительно подготовить:

1. Сообщество:
 - Сообщество в соц.сети ВКонтакте (как это сделать - описано в учебнике, урок Чат-боты 1, п.4 Работа с событиями и чатами). 
 - Для сообщества получите ключ доступа, а также скопируйте ID сообщества в адресной строке браузера (адрес вида https://vk.com/club2040999777835, но нам нужны только цифры!!) 
 - Вставьте ключ доступа и ID сообщества в переменные программы (TOKEN и CLUB, строки 8 и 9)
 - Также понадобится ID в vk пользователя, выполняющего роль преподавателя (переменная vk_prepod, строка 10). 

2. Запустите программу (файл main.py). Бот вышел в эфир, теперь с ним можно общаться.
   ВАЖНО! Бот может общаться сразу с несколькими собеседниками, вы можете писать ему с нескольких аккаунтов одновременно.

3. Напишите боту из любого аккаунта, кроме того, от лица которого создавали сообщество (в vk.com существует ограничение: нельзя написать сообщение в сообщество, которое сам создал... хотя это не точно... ).



...:::  ПЕРЕПИСКА С БОТОМ  :::...

4. При первом контакте бот пытается идентифицировать собеседника как студента по его ID в vk. 
	а) Если ID есть в базе данных, бот присылает сообщение, содержащее перечень возможных действий (4 пункта) и клавиатуру для ввода ответа.
	б) Если ID не найден в базе данных студентов, то бот спрашивает у собеседника группу и фамилию.
	   Когда информация о группе и фамилии получена, бот проверяет, есть ли такой студент в базе. 
	   Если в базе находится студент из указанной группы с такой фамилией, то ID собеседника заносится в базу, и в дальнейшем бот будет его узнавать сразу, без повторной идентификации.
	   Если в базе нет такой фамилии в указанной группе, процедура идентификации начинается заново.	
   Чтобы протестировать этот этап, вы можете выбрать любого студента из базы данных (таблица students). 
   После успешной идентификации ваш ID будет занесен в БД под его фамилией (вы сможете удалить свой ID из базы по окончании работы). 

5. Идентифицированному пользователю присылается сообщение с меню, в котором 4 пункта, и клавиатура для ввода ответа. Доступные действия:
	1) Решить задачу
	2) Узнать дату экзамена
	3) Посмотреть успеваемость
	4) Написать преподавателю

     1) Решить задачу
	Бот извлекает из БД список задач (таблица tasks) и присылает в сообщении одну случайную задачу. После этого бот ожидает ответ от пользователя. 
        Любое сообщение от пользователя будет сравнено с правильным ответом, и бот сообщит собеседнику, правильно он решил задачу, или нет 
        (если пользователь не может решить задачу, он может написать любую чушь, бот ответит, что ответ неправильный).
	После этого все возвращается к меню из п.5

     2) Узнать дату экзамена
	Бот делает запрос к БД: по группе студента определяет ее идентификатор в БД, получает запись из таблицы exams и составляет сообщение с указанием
 	даты, времени и места экзамена (а также будет ли группа тестироваться по ФЕПО). 
	Еще он вычисляет, сколько дней осталось до экзамена, и пишет тревожное сообщение, чтобы студент уже начинал готовиться к экзамену. 
	После этого все возвращается к меню из п.5

     3) Посмотреть успеваемость
	Бот делает запрос к БД по ID студента: определяет его идентификатор в БД, получает все записи по этому идентификатору из таблицы works, 
	и присылает пользователю сообщение с перечнем зачтенных преподавателем лабораторных работ. 
	После этого все возвращается к меню из п.5
	
     4) Написать преподавателю
	Сообщение пользователя вместе с попутной информацией (группа, фамилия студента) отправляется тому пользователю, ID которого указано в переменной vk_prepod. 
	Дальше преподаватель может пообщаться со студентом от своего лица или от лица бота.



...:::  СОСТАВ ПРОЕКТА  :::...

Список файлов в папке проекта:
	IIT_1kurs.db		Файл с базой данных
	main.py			Файл с программой (запускать отсюда) 
	group_keyboard.json	Файл, содержащий клавиатуру, которая посылается пользователю для выбора группы
	keyboard4.json		Файл, содержащий клавиатуру с цифрами "1", "2", "3", "4", для выбора действия
	vk_config.v2.json	Конфигурационный файл для работы с VK API
	requirements.txt 	Список необходимых библиотек
	readme.md		Файл, который вы читаете сейчас

База данных IIT_1kurs.db содержит таблицы:
	exams		Таблица с датами экзаменов в группах
	groups		Список всех групп на потоке
	students	Общий список всех студентов на потоке
	tasks		Тренировочные задачи по теории
	vedomost	Сводная таблица, где отмечено, какой студент (по id студента) какую работу сдал (по id работы)
	works		Перечень всех лабораторных работ курса "Информатика"
